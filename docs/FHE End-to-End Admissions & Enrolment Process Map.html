3 ONLINE ENROLMENT: 13-Step Registration Form
▼
1
Account Creation

Email-based account with secure password setup. Verification email sent to confirm identity.

2
SSO Authentication

Keycloak integration for single sign-on. Role-based access control provisioning.

3
Personal Details

Full name, date of birth, nationality, gender. Validation against identity documents.

4
Contact Information

Primary email (often existing applicant email), phone number. Used for all future communications.

5
Permanent Address

Home address for student records. Required for council tax, voting register, local authority enquiries.

6
Term-time Address

Campus/accommodation address or family address during study. Can be same as permanent.

7
Emergency Contact

Name, relationship, phone number. First contact for emergencies during study.

8
Additional Information

Disability disclosure (ACCES), visa status for international, care experienced status.

9
Previous Education

Schools attended, qualifications obtained (GCSEs, A-levels, degrees, professional certs).

10
Fee Status Declaration

Confirmation of Home/EU/International status. Self-funded vs SLC-funded indication.

11
Document Upload

ID copy, qualification certificates, passport scan (intl), visa documentation. Stored in MinIO.

12
UKVI Compliance Check

International students: Right to study verification, visa type validation, point scoring review.

13
Declaration & Submission

Signature of T&Cs, data accuracy confirmation. Form locked after submission. Student ID generated (STU-2026-XXXX).

🎯 Key Decision Points

  • Document Sufficiency: All docs must be clear and valid before progression
  • UKVI Compliance: International students cannot progress without passing checks
  • Fee Status Accuracy: Determines funding pathway and fee band calculation

4 POST-ENROLMENT PROCESSING: 12-Stage Workflow Engine
▼
1
Eligibility Check

Validate qualifications meet programme entry requirements. Check for gaps or missing evidence.

2
Identity Verification

ID document authenticity check. Match submitted personal data against government ID.

3
Fee Assessment

Determine fee band (Home £9,250, International rate-specific). Create student record with fee liability.

4
SLC Confirmation (Optional)

For students eligible: Send to Student Loans Company for means-testing & loan approval. Skipped for international/self-funded.

5
Document Upload Verification

Final check all required documents received and stored. Flag any missing items for student remediation.

6
Programme Selection Confirmation

Lock-in student's programme enrolment. Create `student_programmes` record linking student to course.

7
Module Registration

Student selects individual modules. System validates credit range (60-240 for fulltime, 30-120 for part-time).

8
Accommodation Preference (Optional)

Student indicates halls vs private housing preference. Allocation managed by accommodation team if applicable.

9
IT Account Provisioning

VLE (Moodle) account creation, university email provisioning, Microsoft Teams enrolment. SSO integrated.

10
Library Registration

Library system provisioning. Physical & digital resource access enabled. Borrowing privileges activated.

11
Student Card Request

Trigger ID card design/printing. Campus access & library integration. Delivery before Welcome Week.

12
Enrolment Complete

All workflow stages passed. Status set to FULLY_ENROLLED. Student now visible in all system portals.

🎯 Key Decision Points

  • Eligibility Failure: Hold enrolment, contact student for missing qualifications or evidence
  • Identity Mismatch: Manual review required before progression
  • SLC Eligibility: Different pathway for international & self-funded students
  • Module Credits: System rejects out-of-range selections automatically

⚙️ Technical Architecture

Data Flow Overview

Application → Admission: Applicant submits form → PostgreSQL stores data → n8n triggers auto-triage → Staff review via portal → Offer decision stored → n8n sends email notification

Pre-Enrolment: Offer accepted → ATR created → n8n auto-comms triggered → Documents uploaded to MinIO → Pre-enrolment pipeline tracked in Redis → Status updates stored in PostgreSQL

Online Enrolment: Student form submission → React frontend → Express.js API → PostgreSQL persistence → Documents to MinIO → Student ID generated → Redis caching for real-time progress

Post-Enrolment: Form complete → n8n orchestrates 12-stage workflow → Each stage validates & updates PostgreSQL → External integrations triggered (SLC, IT, library) → Final status set to FULLY_ENROLLED

Active Study: Student portals consume API endpoints → Real-time dashboard from Redis cache → Moodle VLE delivers teaching → Attendance/engagement data feeds back to PostgreSQL → Analytics dashboards for staff decision-making

🎯 Critical Decision Points & Branching Logic
▼

Decision 1: Home/EU vs International Student

  • Determined: Phase 2 (Pre-Enrolment)
  • Impact: Affects CAS issuance, fee band, visa pathway
  • Home/EU Path: Skip CAS, standard fee £9,250/year
  • International Path: CAS letter, international fee rate, UKVI compliance

Decision 2: Offer Type (Conditional vs Unconditional)

  • Determined: Phase 1 (Admissions)
  • Impact: Acceptance conditions, results day action required
  • Conditional: Student must achieve grades before acceptance valid
  • Unconditional: Immediate acceptance, no grade dependency

Decision 3: DBS/Clearance Required?

  • Programmes Affected: Nursing, teaching, healthcare, social work
  • Timing: Initiated Phase 2, completes before full enrolment
  • If Fail: Offer withdrawn, student notified, enrolment blocked
  • If Pass: Clearance noted, full enrolment proceeds

Decision 4: Fee Funding (SLC vs Self-Funded vs Sponsored)

  • Determined: Phase 3 (Online Enrolment) & Phase 4
  • SLC Path: Student loans company assessment, means-testing
  • Self-Funded: Student/family payment, skip SLC
  • Sponsored: Employer/scholarship funding, special terms

Decision 5: Full-Time vs Part-Time Study Mode

  • Credit Range: Full-time 120 credits/year, part-time 30-60
  • Impact: Fee calculation, student status, assessment timetable
  • Module Selection: System enforces minimum/maximum credit validation
  • SLC Eligibility: Part-time may affect loan eligibility

Decision 6: Accommodation (Residential vs Commuter)

  • Timing: Collected Phase 2, allocated Phase 4
  • Halls of Residence: University-managed accommodation
  • Private Housing: Off-campus arrangements, student sourced
  • Commuter: No accommodation required, home-based study

🔄 Exception Handling & Hold States

  • → Missing Qualifications: Enrolment held until student provides evidence or completes qualification.
  • → Failed Identity Check: Manual review triggered. Student may need to resubmit documents or attend verification appointment.
  • → UKVI Compliance Failure: International student cannot progress. Immigration services notified, offer may be withdrawn.
  • → DBS Failure: Conditional offer withdrawn for regulated programmes. Student notified and offered alternative if available.
  • → SLC Ineligibility: Student must self-fund or seek sponsorship. Fee payment plan adjusted accordingly.
  • → Withdrawal During Enrolment: Student can withdraw at any phase. Status set to WITHDRAWN, offer cancelled, communications sent.
