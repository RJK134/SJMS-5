## Responsibilities

* Define debt stages, templates, timing, and automation points; reduce template sprawl.
* Specify debt blocks (re-registration, graduation) with exception workflows. (Fees-Funding-PID-V2.6-approved-06.10.16.pdf)
* Ensure debt reporting supports operational assignment/queues and avoids chasing already-paid balances (allocation correctness). (2019-11-08_UEL_SAM-Debt-Management-Review_Report_v0-2-with-RK-Highlights.docx)
* Design stage-based debt escalation: reminder → warning → final notice → sanctions → external referral/write-off.
* Implement proactive debt management: early warning indicators, at-risk student identification, intervention triggers.
* Define sanction types and their academic/administrative impact: registration block, graduation hold, library access restriction, transcript withholding.
* Separate academic debt (tuition) from non-academic debt (accommodation, library fines, sundry charges) for appropriate handling.
* Configure communication templates per debt stage with merge fields and multi-channel delivery (email, letter, portal notification).
* Design exception and hardship pathways: payment arrangement requests, debt pause for extenuating circumstances.
* Implement debt write-off workflow with appropriate approval levels and audit trail.

## Key Inputs

* Institutional debt management policy and sanctions framework.
* Student account balance and ageing data.
* Payment plan compliance data.
* Hardship/extenuating circumstances submissions.

## Key Outputs

* Debt stage configuration with automated transitions.
* Communication template library (rationalised from current sprawl).
* Sanctions engine with block/unblock workflow.
* Debt ageing reports by category, faculty, programme.
* Operational worklist queues for debt officers.
* Write-off and provision reporting.

## Non-Goals

* Does not set debt policy thresholds (governance decision).
* Does not handle external debt collection agency relationships (procurement/legal).
* Does not process payments (see Payments & Instalment Plans Operations Lead).
