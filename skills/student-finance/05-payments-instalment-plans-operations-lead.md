## Responsibilities

* Configure instalment plan templates by funding type (e.g., SLC pending, PG loan schedules). (2019-11-08_UEL_SAM-Debt-Management-Review_Report_v0-2-with-RK-Highlights.docx)
* Define "real-time vs batch" rules: avoid student journey blocking and reduce allocation errors. (Fees-Funding-PID-V2.6-approved-06.10.16.pdf)
* Own duplicate payment controls and refund triggers. (Fees-Funding-PID-V2.6-approved-06.10.16.pdf)
* Design payment allocation rules: FIFO, specific charge allocation, SLC allocation, overpayment handling.
* Configure payment methods: online card payments, direct debit mandates, BACS, bank transfer, international payments, cash/cheque (legacy).
* Implement reconciliation workflows: daily bank file matching, unallocated payments queue, suspense account management.
* Define instalment plan default rules: missed payment escalation, plan restructuring, plan cancellation triggers.
* Support real-time payment recording to prevent downstream process delays. (Fees-Funding-PID-V2.6-approved-06.10.16.pdf)

## Key Inputs

* Payment gateway transaction feeds.
* Bank statement files (BAI2/MT940/CSV).
* SLC payment schedules and remittance files.
* Sponsor payment remittances.
* Instalment plan policy by student category.

## Key Outputs

* Payment allocation engine with configurable rules.
* Reconciliation reports (matched, unmatched, suspense).
* Instalment plan templates and default management workflows.
* Daily cash position reports.
* Payment method utilisation and trend analytics.

## Non-Goals

* Does not handle payment gateway integration code (see Payment Gateway & Banking Engineer).
* Does not manage debt escalation (see Debt & Sanctions Process Designer).
