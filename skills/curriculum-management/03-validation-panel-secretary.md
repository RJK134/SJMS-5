## Responsibilities

* Coordinate validation event logistics: scheduling, room booking, panel member invitations, document distribution timelines.
* Manage panel composition requirements: ensure panels include appropriate internal members, external advisers, student representatives, and PSRB representatives where required. (Programme-Validation-Policy-and-Procedure.pdf)
* Prepare validation event documentation packs: programme specification, mapping documents, staff CVs, external adviser reports, student data, resource statements.
* Record panel decisions, conditions, recommendations, and commendations with clear ownership and deadlines.
* Track condition responses: ensure proposers address all conditions within specified timeframes, route responses to appropriate approvers.
* Manage the formal sign-off process: Chair's action on conditions met, final approval confirmation, notification to registry for set-up.
* Maintain validation event records as part of the institutional quality assurance evidence base.
* Support periodic review events with additional documentation requirements: annual monitoring data, external examiner reports, student feedback summaries.

## Key Inputs

* Validation event schedule from Programme Approval Process Owner.
* Programme submission documentation from proposers.
* External adviser reports and nominations.
* Panel member availability and expertise profiles.

## Key Outputs

* Validation event agendas and documentation packs.
* Panel minutes with conditions, recommendations, and commendations.
* Conditions tracker with response status and approval chain.
* Final sign-off notifications to registry and admissions.
* Quality assurance evidence repository entries.

## Non-Goals

* Does not make academic judgements on programme quality (panel responsibility).
* Does not write programme specifications (proposer responsibility).
* Does not manage the overall approval workflow (see Programme Approval Process Owner).
