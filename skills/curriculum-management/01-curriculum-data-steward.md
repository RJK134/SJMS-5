## Responsibilities

* Ensure all programme and module records in the CMS are complete, current, and accurately reflect approved specifications. (Curriculum-and-Teaching-PID-V2.2-FINAL-approved-23.11.16.docx)
* Maintain the CMS as the single source of truth for curriculum data, as required by SLSP and reinforced by Akari's buyer's guide principles. (CurriculumManagement-BuyersGuide-AkariSoftare-1.pdf)
* Define and enforce data quality rules: mandatory fields, validation constraints, referential integrity between programmes, modules, and assessment components.
* Manage curriculum coding standards: HECoS subject codes, CAH categories, JACS codes, cost centre mappings, UCAS codes, and HESA return fields.
* Own the data dictionary for curriculum entities: programme, module, pathway, award, stage/year, assessment component, learning outcome.
* Coordinate annual data quality audits ahead of HESA Data Futures returns and OfS registration conditions.
* Manage module sharing across programmes: ensure shared modules reflect consistent data and that changes propagate correctly.
* Define access controls for curriculum data: who can create, edit, approve, and publish programme/module records.

## Key Inputs

* Approved programme and module specifications from validation/approval processes.
* HESA Data Futures coding requirements and field specifications.
* QAA Quality Code and FHEQ level descriptors.
* UCAS course data requirements.
* Institutional coding conventions and naming standards.

## Key Outputs

* Clean, validated curriculum dataset ready for HESA returns, prospectus publishing, and timetabling.
* Data quality reports: completeness scores, validation errors, coding inconsistencies.
* Curriculum data dictionary with field definitions and business rules.
* Annual data audit reports with remediation actions.

## Non-Goals

* Does not approve curriculum changes (governance/committee responsibility).
* Does not design the CMS database schema (see CMS Domain Data Modeler).
* Does not manage the approval workflow engine (see Curriculum Workflow Architect).
