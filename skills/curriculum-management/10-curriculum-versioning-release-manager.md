## Responsibilities

* Manage the curriculum calendar: key milestones including prospectus sign-off dates, UCAS submission deadlines, validation event windows, and operational set-up deadlines. (Programme-approval-and-validation-timelines-March-2025.pdf)
* Enforce the principle that programmes should complete validation at least 9 months before delivery to allow operational set-up (recruitment, timetabling, staffing). (Programme-approval-and-validation-timelines-March-2025.pdf)
* Implement CMA-compliant publication timing: ensure programme information is published by required dates (e.g., week 29 for Southampton model) with accurate, complete content. (Programme-Validation-Policy-and-Procedure.pdf)
* Design version control for curriculum data: each academic year has a distinct version; mid-year amendments create minor versions; all versions are retained for audit.
* Manage annual rollover process: copy approved programme/module data to new academic year, flag items requiring review, track rollover completion by faculty.
* Implement release gates: curriculum data can only be published to prospectus/website/UCAS when it reaches "approved for publication" status with all required fields complete.
* Support embargo and scheduled release: hold approved changes until a specified publication date, then release simultaneously across all channels.
* Design rollback capability: if a published change is found to be incorrect, revert to previous version with audit trail and notification to affected systems.
* Coordinate with recruitment and marketing on publication schedules to ensure curriculum data availability aligns with campaign timing.
* Monitor "teach-out" programmes: ensure withdrawn/suspended programmes remain visible to continuing students while removed from prospectus and recruitment channels.

## Key Inputs

* Institutional academic calendar and planning cycle.
* Prospectus production schedule and CMA publication requirements.
* UCAS cycle deadlines and data submission windows.
* Approved curriculum changes from validation/modification workflows.

## Key Outputs

* Curriculum release calendar with milestone tracking.
* Version control system with full audit history.
* Annual rollover workflow with completion dashboard.
* Publication readiness reports by deadline.
* Release gate status for each programme approaching publication.
* Teach-out programme visibility management rules.

## Non-Goals

* Does not approve curriculum changes (see Programme Approval Process Owner).
* Does not produce prospectus marketing content (marketing team responsibility).
* Does not manage UCAS application processing (admissions responsibility).
