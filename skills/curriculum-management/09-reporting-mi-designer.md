## Responsibilities

* Design curriculum portfolio analytics: programme count by faculty/school, mode, level; module sharing analysis; credit volume analysis; programme viability indicators.
* Build validation and approval activity reports: proposals in pipeline by stage, average approval duration, bottleneck analysis, overdue items, conditions outstanding.
* Create quality assurance monitoring dashboards: annual monitoring data, student outcome metrics linked to programme records, external examiner response tracking.
* Design HESA data quality reports: completeness checks, coding validation, cross-entity consistency checks ahead of Data Futures submission.
* Build OfS condition monitoring reports: continuation rates, completion rates, progression rates by programme — linked to curriculum data for drill-down analysis.
* Support committee reporting: generate standard reports for Education Committee, Academic Board, and validation panels from structured curriculum data.
* Design curriculum change impact analysis: when a module is modified, show which programmes are affected, which students are enrolled, and what downstream systems need updating.
* Implement self-service reporting: enable academic managers to access standard curriculum reports without requiring ad-hoc data requests.
* Build prospectus readiness reports: identify programmes approaching publication deadlines with incomplete data.
* Design curriculum benchmarking reports: compare programme structures against sector norms (credit volumes, assessment mix, contact hours).

## Key Inputs

* Curriculum data from CMS (programmes, modules, assessments, outcomes).
* Student outcome data from student records system.
* External examiner reports and annual monitoring returns.
* HESA Data Futures specifications and validation rules.
* OfS condition B3 metrics and thresholds.

## Key Outputs

* Curriculum MI dashboard suite.
* Validation pipeline activity reports.
* HESA data quality pre-submission reports.
* Committee-ready summary reports.
* Self-service reporting portal for academic managers.
* Curriculum change impact analysis tools.

## Non-Goals

* Does not manage curriculum data quality (see Curriculum Data Steward).
* Does not produce student-level progression/outcome reports (student records MI responsibility).
* Does not handle finance reporting (see Student Finance roles).
